Se você é interessado em um Optimal Cube Solver na Métrica de Quarto de Volta que é executado na linha de comando em LINUX e WINDOWS, você pode baixar o código-fonte C documentado aqui. Uma versão já compilada para Windows está disponível aqui. O programa também aceita o formato de arquivo do Cube Explorer, para que você possa gerar seus cubos no Cube Explorer e alimentá-los para este programa.
Bruce MacKenzie portou este programa de linha de comando para ser executado em um MAC (nomen est omen). Você pode baixar o programa aqui.

Para fins de demonstração, escrevi um pacote Java que implementa o algoritmo de duas fases em sua forma mais simples, sem reduções de simetria.

O pacote org.kociemba.twophase, o código-fonte e os javadocs correspondentes estão incluídos no arquivo twophase.jar . O pequeno programa Java GUI_example.jar (Versão 2009.02.16), que é um arquivo jar executável mostra um exemplo de como usar o pacote.

As tabelas nesta implementação levam apenas cerca de 5 MB e são geradas em segundos. No entanto, a rotina do pacote resolveu cerca de 26000 cubos aleatórios/hora se o comprimento máximo de manobra fosse definido para 21 movimentos e cerca de 800 cubos aleatórios/hora se fosse definido para 20 movimentos de comprimento máximo.

Você pode usar este pacote gratuitamente, mas você deve incluir uma linha de crédito apropriada.

Por último, mas não menos importante, implementei o algoritmo de duas fases em um pacote Mathematica. O código é executado muito lento, mas também é muito curto. Pode ser interessante do ponto de vista teórico.

A função de editor interativo no pacote precisa de pelo menos a versão 6.0 do Mathematica.
