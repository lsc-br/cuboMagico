Introduo: CUBOTino Autnomo: Pequeno, Impresso em 3D, Rob Cubo Mgico (Raspberry Pi Zero 2, PiCamera)

Por AndreaFaveroandrea.favero71@gmail.com
Mais pelo autor:
Sobre: Nasci na Itlia em 1971, e desde poucos anos moro com minha famlia na Holanda. No tempo livre eu gosto de construir "dispositivos inteis", um pr-teste para aprender codificao e teoria de controle Mais sobre AndreaFavero

Esta a "verso Top" da srie CUBOTino, um projeto de rob solucionador de cubos da Rubik destinado a ser muito pequeno, simples, totalmente impresso em 3D e escalvel.

Este claramente no um rob rpido, pois os objetivos so simplicidade e custo-benefcio.

A "Verso Base", caracterizada por menores custos e menor complexidade, est disponvel em: https://www.instructables.com/CUBOTino-a-Small-Simple-3D-Printed-Inexpensive-Rub/

Esta instruo refere-se verso Top; As principais caractersticas so:

  • O rob totalmente autnomo.
  • PiCamera e Raspberry Pi Zero 2 so usados para a parte de viso computacional.
  • Kociemba solver usado, para obter uma soluo quase ideal.
  • Tudo codificado em Python.
  • Exibir para feedback sobre a ao do rob e o progresso da resoluo.
  • Alm da capa PCB, todas as outras partes impressas em 3D so as mesmas da verso Base, alm de duas novas peas para suportar a cmera.
  • O tempo de resoluo tipicamente de 90 segundos; So necessrios 30 segundos para a deteco do status do cubo e 60 para resolver o cubo. Este tempo uma vez que o rob otimizado, de acordo com os filmes postados

Ao excluir os cubos de Rubik de auto-resoluo e os robs que exigem cubos modificados para agarrar, este um dos menores robs de cubo de Rubik que j vi at agora.

O nome deste rob CUBOTino: Alm do bvio be+roBOT, "ino" o sufixo italiano que significa pequeno..... um "pequeno" toque que lembra minhas origens e as dimenses muito pequenas do rob.

Dica: No Passo 2 destes Instructables est embutido um arquivo pdf "How_to_build_ ..._yyyymmdd.pdf", de cerca de 150 pginas, contendo todos os detalhes necessrios para construir, programar, sintonizar, troubbleshoot, aprender, etc.


Crditos para:

  • Sr. Kociemba, por seu solver, e a verso python que ele disponibilizou para a comunidade.
  • Yheneaul por simplificar a instalao e a Vanblok por projetar a PCB.
  • Todos vocs que deram feedback, possibilitando a melhoria do projeto.

Suprimentos

O custo do material tem sido de cerca de 100 euros para mim, mas desta vez eu usei principalmente entregas bastante rpidas ....

Raspberry Pi Zero 2 no pode ser facilmente encontrado, e altamente caro devido a uma terrvel escassez de material;

Edit 29/12/2022: No final de dezembro de 2022 o Raspberry Pi Zero W (no Zero 2) est novamente disponvel: eu poderia obter uma pea por 18 na loja oficial na Holanda. Raspberry Pi Zero W tem sido usado por vrios Makers para este projeto ... e em breve vou testar o meu.


As peas mais caras do projeto so:

  • Raspberry Pi Zero 2 WH (tambm funciona com Raspberry Pi Zero W, levando mais tempo de computao)
  • carto microSD 16Gb Class10
  • Picmera (V1.3)
  • Cabo flexvel (PiCamera - Raspberry Pi Zero)
  • Dois servos (TD-8325MG, 180deg 25Kg metal, com brao de metal "25T")
  • Display (1,77" TFT, SPI)
  • cerca de 500 gramas de filamento
  • Led de 3W, com driver integrado
  • Sensor de toque cativo
  • Regulador de tenso abaixador
  • Placas de breakout micro-usb (ou placa de breakout USB-C)

Alguns pequenos componentes eletrnicos (placa prottipo, capacitores, conectores, etc), e parafusos.

Para a fonte de alimentao so necessrios dois carregadores de telefone (al pelo menos 1x1A e 1x2A), e cabos relacionados com conector microUSB. Como alternativa, fonte de alimentao 1x3A com conector USB-C (o Raspberry Pi oficial recomendado).

Nas instrues esto listadas possveis alternativas de servos.

No arquivo de Components_list_Cubotino_Top.pdf anexo a lista detalhada, incluiu o link para as lojas que eu usei ... e custo relacionado.

Step 3: Make the Connection Board

The connections board connects all the devices (servos, display, touch button and led breakout board) to the Raspberry Pi.

It is not complex, but it requires more time and attention compared to the one for the Cubotino Base version; In case of mistakes, it will be difficult to de-solder the 2x20 header, perhaps better to buy few spares in advance.

The board is made out of a prototype board (two sides), mainly by connecting different connectors and few capacitors.

Better to use insulated wire, when crossing other lines, even if these are located on the other board side.

It is possible to order the pcb, therefore limiting the effort and possible mistakes; Components purchase and soldering still need to be done.

Gerber files made available by Chad; V1.6 version is stored at GitHub: https://github.com/AndreaFavero71/cubotino/tree/main/connections_board/gerber

Additional info at "How_to_make_ .... .pdf" doc, also on how to download the Gerber files from GitHub

Step 5: Set the Servos Angle for Correct Assembly

Set both the servos to the mid angle; This can be done via below steps:

  1. connect the servos to the Connection board
  2. enter the robot code folder, by typing 'cd cubotino/src' from the root
  3. activate the virtual environment, by typing 'source .virtualenvs/bin/activate'
  4. set the servos to the mid angle, by typing 'python Cubotino_T_servos.py --set 0'

On the servo for the Top_cover: Connect the T25 arm oriented as per picture.

This also is a good moment to verify if the rotation range of the received servos, as well as the Pulse Width characteristics.

More info in the annex "How_to_make_ .... .pdf".

Step 6: 3D Prints

There are a total of 13 parts to be 3D printed, for about 500 grams (when PETG is used).

All parts have been designed to be printed:

  • on printers with at least 200x200mm printing plate
  • without supporting the overhangs


More info, like the suggested print orientation, in the annex "How_to_make_ .... .pdf".

Note: If you want to upgrade Cubotino Base version, the 'old' PCB_cover won't be re-usable and few new more parts have to be printed.

From 6th August 2022, In case you'd like to modify the parts/project, the step files at stored at GitHub: https://github.com/AndreaFavero71/cubotino/tree/main/stp

Step 7: Assemble the Robot

All parts are connected by screws.

Some little hot glue to keep the touch button in position.

Add four self-adhesive rubber feet underneath the base.

In the annex "How_to_make_ .... .pdf", the assembly steps and details are provided.

Hi Matze, the code has this functionality.
It is explained on chapter 19 "How to operate the robot", point 9 "Raspberry Pi shut down".
The way I prefer is the one that doesn't require any PC or phone connection with the robot; This can be achieved by setting "Rpi shut down via Touch button" indicated at chapter 21.
Once this is set, and after a reboot, the raspberry Pi can be shut off by keeping the touch button "pressed" very long (about 6 seconds): Keep pressed until "SHUT DOWN" appears on the screen, then the button can be released; Wait until the LED goes off, plus 10 seconds, then you can safely unplug the power supply.
I know the instructions are bit extensive, and not necessary following a logis order for everybody, yet the search function can be used with keywords. By using the word "Shutdown" you'll land at on chapter 19 "How to operate the robot", point 9 "Raspberry Pi shut down" :-)

Hi Matze,
it looks like your touch sensor board resets its "ON" status in a smaller time than the code is currently considering to start the quit procedure. You might try to measure this time by watching at the touch sensor led while monitoring the time with a stop watch.
Anyhow, to solve this issue, it is necessary to reduce the "quit_time" parameter, located at "Cubotino_T_settings.txt"; The default value is 4.5 (seconds) and you might try with 3.5 or less.
The "quit_time" parameter identifies a delay from the moment the touch_button is pressed and the starting of the quitting procedure; A larger "quite_time" prevent an unwanted quitting when the finger sleeps over the button....
In case you need to reduce a lot the "quit_time" (I don't think is really needed), you might also reduce "warn_time", for instance from the default value of 1.5 to 1.0 (seconds).

1) Connect to the Raspberry Pi via VNC viewer
2) Edit crontab (sudo nano crontab -e)
2a) Comment the row with the low screen resolution
2b) Uncomment the row with the high screen resolution
2c) Reboot (sudo reboot)
3) Connect to the Raspberry Pi via VNC viewer
4) Enter the project folder (cd cubotino/src)
5) Activate the venv (source .virtualenvs/bin/activate)
6) Launch the GUI (python Cubotino_T_servos_GUI.py)

Hi Matze,
in order to see the complete GUI window it is indeed necessary to change the VNC resolution.
This is explained at Step10 of the "Setting up raspberry Pi" chapter, yet it will be more clear on the next instructions release (see annex picture as reference).
Steps:
1) Connect to the Raspberry Pi via VNC
2) Edit the crontab (sudo crontab -e)
3) If it is the first time the crontab is edited, you have to choose which editor; nano is more than ok for this little file, therefore press "1" and Enter.
4) Comment out the row that ends with "1280x720"
5) Uncomment the row that ends with "1920x1080"
6) Save and exit (Ctrl X, Y, Enter)
7) Reboot (sudo reboot)
8) Connect to the Raspberry Pi via VNC
9) Enter the project folder, activate the venv, launch the GUI
